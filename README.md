# AI Scrum Starter

Run your AI assistant like a Scrum team. You're the Product Owner; you direct
specialized agents — an **Architect** to plan, a **Scrum Master** to orchestrate
a team of sub-agent developers — through a sprint.

### New here? Open `GETTING_STARTED.md` and follow the five steps.

## What's in this folder

| File | What it's for |
|------|---------------|
| `GETTING_STARTED.md` | Five-step quickstart for your first project. Start here. |
| `CLAUDE.md` | The role definitions. Loads automatically when you open Claude Code in this folder. |
| `SCRUM_MASTER_KICKOFF.md` | The prompt that turns a Claude Code session into your Scrum Master. |
| `templates/VISION_TEMPLATE.md` | Fill this in for your own project. |
| `templates/VISION_EXAMPLE.md` | A finished VISION, for reference. |
| `examples/backup_intakes.md` | Three ready-made starter projects if you don't have one in mind. |
| `examples/ISSUE_REPORT_example.md` | What a sub-agent surfaces when it hits a problem. |
| `reference/HOW_THIS_RELATES_TO_HERMES.md` | Where this sits in the agent-harness landscape. |

## The one idea

The **harness** — the instructions, constraints, and roles around the model — is
where the leverage is, not the model itself. These files *are* that harness. The
discipline upstream of the prompt is the whole game.
